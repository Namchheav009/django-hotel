# Django Hotel Management System

This is a simple hotel management web application built with Django. It provides functionality for guests to browse rooms, services, make reservations, manage their cart, and book services. Administrators can manage rooms, services, bookings, users, and view reports.

## Features

- Guest-facing pages:
  - Home page
  - Room listing and details
  - Service listing and booking
  - Reservation management (cart, checkout)
  - User authentication (login/register)
  - Profile management and booking history
  - Reviews and feedback
  - Contact form

- Admin panel:
  - Manage rooms, categories, prices, images
  - Manage services and service bookings
  - Manage user accounts and profiles
  - View reports and reservations
  - Handle payments and categories

## Technologies

- Python 3.11+ (tested with 3.13)
- Django 4.x
- SQLite (default development database)
- Bootstrap 5 for styling
- Font Awesome
- SweetAlert2 for alerts

## Setup Instructions

1. **Clone the repository**
   ```bash
   git clone <repo-url> django-hotel
   cd django-hotel/Hotelproject
   ```

2. **Create and activate a virtual environment**
   ```bash
   python -m venv .venv
   # Windows (PowerShell)
   & .venv\Scripts\Activate.ps1
   # macOS / Linux
   source .venv/bin/activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Run database migrations**
   ```bash
   python manage.py migrate
   ```

5. **Create a superuser (admin)**
   ```bash
   python manage.py createsuperuser
   ```

6. **Collect static files (if needed)**
   ```bash
   python manage.py collectstatic
   ```

7. **Run the development server**
   ```bash
   python manage.py runserver
   ```

8. **Access the application**
   - Guest UI: http://127.0.0.1:8000/
   - Admin panel: http://127.0.0.1:8000/admin/

## Notes

- The project uses `settings.py` with default SQLite configuration. Adjust `DATABASES` for production.
- Static assets are located under `hotel/static/`. Templates are in `hotel/templates/`.
- Media files (room images, service images) are stored in the `media/` directory.

## Development Tips

- Use `python manage.py runserver --settings=Hotelproject.settings` if custom settings needed.
- Add required environment variables for secret keys and production settings.
- Remember to set `DEBUG = False` in production and configure allowed hosts.
